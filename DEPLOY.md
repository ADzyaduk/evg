# Перед деплоем

## Проверки (одной командой)

```bash
pnpm run predeploy
```

Выполняет по порядку: `lint` → `typecheck` → `build`. Если всё ок — можно деплоить.

## По отдельности

- **Линт:** `pnpm run lint` (стиль и часть ошибок можно автофиксировать: `pnpm run lint -- --fix`)
- **Типы:** `pnpm run typecheck`
- **Сборка:** `pnpm run build`
- **Локальный просмотр продакшен-сборки:** `pnpm run preview`

## Переменные окружения на хосте

Для формы обратной связи (отправка в Telegram) задайте:

- `NUXT_TELEGRAM_BOT_TOKEN` — токен бота
- `NUXT_TELEGRAM_CHAT_ID` — ID чата, куда слать заявки

Без них форма вернёт ошибку при отправке.

## Уже настроено

- **SSR** включён (SEO, превью в соцсетях).
- **Devtools** отключены в продакшене.
- **Prerender** для главной (`/`) — страница собирается в статику при билде.
- Шрифты только локальные, без внешних запросов.

## Производительность

- Переходы между страницами и анимации появления уже есть; при желании можно добавить пререндер других маршрутов в `nuxt.config.ts` → `routeRules` (например, `/blog`, `/kontakty`).
- Изображения в `public/` отдаются как есть; для сжатия и форматов next-gen можно позже подключить `@nuxt/image` и заменить `<img>` на `<NuxtImg>`.
